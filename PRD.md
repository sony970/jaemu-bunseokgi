# PRD: 재무분석기 (DART 기반 기업 재무제표 비교 도구)

## 1. 개요
회사명을 입력하면 금융감독원 전자공시시스템(DART)의 공시 데이터를 조회하여, 여러 기업의 최근 3개년 재무제표를 비교하고 엑셀 파일로 저장해주는 CLI 도구.

## 2. 배경 및 문제 정의
- 기업의 재무제표를 비교하려면 DART 전자공시에서 회사마다 개별적으로 사업보고서를 찾아 열람해야 하고, 원하는 지표(매출액, 영업이익 등)를 수작업으로 옮겨 적어야 함
- 여러 회사를 나란히 비교할 수 있는 표 형태의 자료가 즉시 필요한 경우가 많음(경쟁사 분석, 투자 검토 등)
- 이를 자동화해 회사명 입력만으로 비교 가능한 엑셀 산출물을 즉시 얻는 것이 목표

## 3. 목표
- 회사명(한글) 입력만으로 DART 공시 상 정확한 기업을 식별
- 최근 3개년 핵심 재무지표를 자동 추출
- 여러 회사를 한 눈에 비교할 수 있는 엑셀 파일로 저장
- 별도 유료 API나 수작업 데이터 수집 없이, 무료 공공 데이터(OpenDART)만으로 동작

## 4. 대상 사용자
- 기업 재무 비교가 필요한 개인 사용자(투자자, 취업준비생, 애널리스트 등)
- 별도 개발 지식 없이 커맨드 한 줄로 결과물을 받고 싶은 실무자

## 5. 범위

### 5.1 In Scope (v1 — 구현 완료)
| 항목 | 설명 |
|---|---|
| 회사 검색 | DART `corpCode.xml` 전체 목록 캐싱 후 회사명으로 매칭(정확일치 → 부분일치, 상장사 우선) |
| 재무제표 조회 | `fnlttSinglAcntAll.json` API로 최근 사업보고서(연간) 기준 당기/전기/전전기 3개년 조회 |
| 연결/별도 재무제표 폴백 | 연결재무제표(CFS) 우선 조회, 데이터 없으면 별도재무제표(OFS)로 자동 재시도 |
| 핵심 지표 추출 | 매출액, 영업이익, 당기순이익, 자산총계, 부채총계, 자본총계 |
| 연도 자동 탐색 | `--year` 미지정 시 최신 사업보고서가 존재하는 연도를 역순으로 자동 탐색 |
| 엑셀 저장 | 비교요약 시트(회사×연도×지표) + 회사별 상세 시트(원본 계정 전체) |
| 에러 처리 | 회사명 미검색 시 후보 제시, API 오류 시 상태코드/메시지 출력 |

### 5.2 Out of Scope (v1)
- 재무비율(ROE, 부채비율, 유동비율 등) 자동 계산 및 해석
- 분기/반기 보고서 비교 (현재는 연간 사업보고서만)
- GUI/웹 인터페이스 (현재는 CLI)
- 3개사 초과 비교
- 비상장사·해외기업 지원
- 데이터 시각화(차트)

## 6. 기능 요구사항
1. 사용자는 회사명 3개를 커맨드라인 인자로 입력한다.
2. 시스템은 각 회사명에 대해 DART 고유번호(corp_code)를 조회한다.
3. 회사명이 여러 후보와 매칭될 경우, 상장사를 우선하고 그 외에는 이름이 가장 짧게 일치하는 후보를 선택한다.
4. 매칭 실패 시 유사 후보명을 최대 5개까지 제시하고 프로그램을 종료한다.
5. 시스템은 각 회사의 최근 3개년(당기/전기/전전기) 재무제표를 1회의 API 호출로 조회한다.
6. `--year` 옵션으로 특정 사업연도를 지정할 수 있다.
7. 시스템은 조회된 계정 목록에서 6개 핵심 지표를 표준화된 이름으로 추출한다.
8. 결과는 `.xlsx` 파일로 저장되며, 회사×연도 비교 요약 시트와 회사별 상세 시트를 포함한다.
9. `--output` 옵션으로 저장 경로를 지정할 수 있으며, 미지정 시 `output/재무비교_YYYYMMDD.xlsx`로 저장한다.

## 7. 비기능 요구사항
- **성능**: 회사 3개 조회 및 엑셀 저장까지 수 초~수십 초 이내 (DART 응답 속도에 의존)
- **캐싱**: 고유번호 전체 목록은 로컬에 캐시하여 7일간 재사용, 매 실행마다 재다운로드하지 않음
- **신뢰성**: DART API 오류(상태코드 000이 아닌 경우)를 사용자에게 명확히 안내
- **보안**: API 키는 로컬 `config.py`에 저장, 외부로 전송하지 않음

## 8. 기술 스택 및 아키텍처
- 언어: Python 3
- 라이브러리: `requests`(API 호출), `pandas`/`openpyxl`(엑셀 생성)
- 데이터 소스: OpenDART API (`corpCode.xml`, `fnlttSinglAcntAll.json`)
- 구조:
  - `dart_client.py`: DART API 통신, 회사 검색, 재무데이터 파싱 담당
  - `main.py`: CLI 인자 처리, 조회 오케스트레이션, 엑셀 생성
  - `config.py`: API 키 보관
  - `cache/corp_codes.json`: 회사 고유번호 캐시

## 9. 성공 기준 (v1)
- 실존하는 국내 상장사 3곳 입력 시 정상적으로 엑셀 파일 생성
- 생성된 엑셀의 수치가 실제 DART 공시 수치와 일치
- 존재하지 않는 회사명 입력 시 적절한 에러 메시지 출력

> v1은 위 기준을 충족한 상태로 구현 및 실데이터(삼성전자/SK하이닉스/LG전자) 검증 완료.

## 10. 리스크 및 제약
- DART API 일일 호출 한도 초과 시(`status=020`) 조회 실패 가능
- 계정과목명이 회사마다 상이하여(예: "매출액" vs "수익(매출액)") 일부 회사는 지표가 비어있을 수 있음
- 비상장사 또는 감사보고서 미제출 기업은 데이터가 없을 수 있음
- 3월 결산 등 12월 결산이 아닌 회사는 연도 라벨이 실제 회계연도와 다르게 표시될 수 있음

## 12. 배포 및 공개 계획
- **GitHub**: 소스코드를 공개/비공개 저장소로 관리. `config.py`에 API 키를 평문으로 저장하는 현재 방식은 저장소에 커밋하면 키가 유출되므로, GitHub에 올리기 전 API 키를 `.env` 또는 환경변수로 분리하고 `.gitignore`에 추가하는 작업이 선행되어야 함.
- **Vercel 배포**: Vercel은 Node.js 기반(Next.js 등) 또는 서버리스 함수 배포에 최적화되어 있어, 현재의 Python CLI 구조를 그대로 올릴 수 없음. 배포를 위해서는 아래 중 한 방향의 전환이 필요:
  1. 백엔드 로직(DART 조회·엑셀 생성)을 Vercel Serverless Function(Python 런타임 또는 Node.js 재작성)으로 이전
  2. 프론트엔드(Next.js 등)에서 회사명 입력 폼을 제공하고, 서버리스 함수가 조회 결과를 반환 (파일 다운로드 형태 또는 웹 화면에 표 렌더링)
  3. `cache/corp_codes.json` 로컬 파일 캐싱 방식은 서버리스 환경(파일시스템 휘발성)에 맞지 않으므로 외부 스토리지(KV, Blob 등) 또는 매 요청 시 재다운로드로 대체 필요
- 위 전환 작업의 구체적인 설계는 별도 논의 후 확정 예정.
